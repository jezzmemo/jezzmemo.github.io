#runloop 本人学习iOS已经多年了，最近在和同事一起讨论的时候这个的时候竟然发现自己对这个问题还是理解的不是很深入，为了更加深入 的学习和理解Runloop，这次我们深入全面的来学习下Runloop. 1.什么叫Runloop: 我们一般的程序都会执行完成后退出，runloop，根据字面的意思就是一直处于运行状态，直到收到某个消息的时候，才退出运行状态。 比如我们的操作系统有点类似的，只要开机就一直处于运行状态，直到关机才处于结束状态，根据这种字面的理解，我们就可以得出iOS的 Runloop是在程序启动的时候就开始运行，程序真正结束的时候才退出。 iOS中有两种方式获取Runloop对象,一种是NSRunLoop，另外一种是CFRunLoopRef，NSRunLoop是CFRunLoopRef的封装，不同的是NSRunLoop 不是线程安全的，CFRunLoopRef是线程安全的。 2.iOS的runloop的细节 首先我们还是不厌其烦的拿出官方的解释图： https://developer.apple.com/library/ios/documentation/Cocoa/Conceptual/Multithreading/Art/runloop.jpg 从图中我们分解出以下的知识点： 1.源(Source)，分成:(input source 和timer source) 2.1 Input source是是异步的形式，消息一般来源与另外一个线程或者另外一个application. Timer是同步的，通过设置执行时间或者重复执行的间. Input sources deliver events asynchronously to your threads. 2.1.1 Input source有两种方式， 一种是Port-base的方式， 另外一种是自定义消息 Port-based 是监听应用的 Mach ports. 这种方式需要添加一个NSMachPort对象到Runloop的Source中， 当Port中有消息发送过来时，会唤醒Runloop来处理消息: - (void)handleMachMessage:(void *)msg { } 自定义消息是监听自定义的事件.如图所示: https://developer.apple.com/library/ios/documentation/Cocoa/Conceptual/Multithreading/Art/custominputsource.jpg 如果想使用自定义的Input Source向Runloop投递消息，需要使用CoreFoundation的API，Foundation的API不提供创建自定义的Input Source，根据文档，使用Input Source需要如下步骤： 创建一个CFRunloopContextRef,实现三个回调函数: void (*schedule)(void *info, CFRunLoopRef rl, CFStringRef mode); void (*cancel)(void *info, CFRunLoopRef rl, CFStringRef mode); void (*perform)(void *info); 通过上面的context创建一个RunloopSourceRef: CFRunLoopSourceRef runLoopSource = CFRunLoopSourceCreate(NULL, 0, &context); 将创建的RunloopSource添加到当前的Runloop中: CFRunLoopRef runLoop = CFRunLoopGetCurrent(); CFRunLoopAddSource(runLoop, runLoopSource, kCFRunLoopDefaultMode); 向RunloopSource发送Signal，唤醒Runloop处理事件 CFRunLoopSourceSignal(runLoopSource); CFRunLoopWakeUp(runloop); 两个不同的地方在于，Port-based是由kernel自动发送信号量的，而自定义消息是由用户手动来控制发送的. 2.Run Loop Modes 3.Run Loop Observers 4.Runloop的流程和时序 参考资料: https://developer.apple.com/library/ios/documentation/Cocoa/Conceptual/Multithreading/RunLoopManagement/RunLoopManagement.html
