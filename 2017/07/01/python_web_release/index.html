部署Flask程序

最近公司需要开启一个实验性的项目，趁此机会，可以从零开始部署一个全新的Flask的工程，是对Flask进一步认识和运用的加强。

在安装公司的Flask程序，我们首先在服务器安装以下程序:

  • Nginx
  • Supervisor

在此我们省略这两个软件的安装过程，随便Google搜索下，即可找到你想要步骤，一般分为源码编译安装和二进制安装方法，剩下的工作就是我们去如何去配置它们.

2.配置Nginx

因为服务器已经安装好了，直接把配置的示例给出来:

proxy_pass http://127.0.0.1:6000，这里Nginx只是作为反向代理服务器，主要作为请求的转发作用和域名绑定，这个6000端口是由gunicorn服务器绑定的.

3.Gunicorn配置

Gunicorn (独角兽)是一个高效的Python WSGI Server,通常用它来运行 wsgi application(由我们自己编写遵循WSGI application的编写规范) 或者 wsgi framework(如Django,Paster),地位相当于Java中的Tomcat。

我们会用supervisor来配置启动和关闭网站，示例如下:

gunicorn使用配置文件的方式来启动，配置里面主要是配置端口和IP,还有多少个Workers,就是多少个进程，以及worker_class是用的同步的方式还是异步的方式来处理请求，在这里我们选择用Gevent异步的方式来处理请求，Gunicorn默认是同步处理请求的，还有就是重写了gunicorn重启的时候执行函数，以及配置出错信息存储的地方。

在启动的时候会遇到各种错误，errorlog在当前目录Var文件夹里error.log文件，错误的信息都比较明显，根据这里的错误依次来解决，我来列举下我遇到的错误:

  • gevent没有此模块
    Gunicorn启动app的时候，提示没有安装gevent模块，在当前虚拟环境下pip install gevent即可

  • upstream prematurely closed connection while reading response header from upstream
    这错误是Flask程序在连接数据库导致超时，Nginx返回502错误.

4.总结

Nginx作为反向代理作为所有请求的前端支点，还有静态文件处理都归Nginx处理.
Gunicorn作为一个Web容器来处理所有的动态请求，Gunicorn本身会启动多个进程来处理.
